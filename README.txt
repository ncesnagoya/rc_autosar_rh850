１．概要

モデルカー制御プログラムはRCカーをベースとしたモデルカーを制御するため
のプログラムであり，TOPPERSプロジェクトから公開されている車載制御向け
のソフトウェアプラットホームであるTOPPERS/APを使用している．


２．プログラム構成

モデルカー制御プログラムは，機能や使用するプラットホームの違いにより次
のバリエーションがある．

1.AUTOSAR OS 対応プログラム
  各機能をタスクとして実現しているプログラム．
  RTE等は使用していない．
  
2.AUTOSAR SW-C 対応プログラム:1ECU
  各機能をランナブルとして実現しているプログラム．
  1個のECUで動作する．
  
3.AUTOSAR SW-C 対応プログラム:2ECU
  各機能をランナブルとして実現しているプログラム．
  2個のECUで動作する．
    
4.AUTOSAR SW-C 対応プログラム:2ECU(機能追加)
  各機能をランナブルとして実現しているプログラム．
  2個のECUで動作する．"AUTOSAR SW-C 対応プログラム:2ECU"に機能を追加し
  ている．
        
5.AUTOSAR SW-C 対応プログラム:2ECU(SW-C移動)
  各機能をランナブルとして実現しているプログラム．
  2個のECUで動作する．"AUTOSAR SW-C 対応プログラム:2ECU(機能追加)"の機
  能配置を変更する．
    
6.AUTOSAR SW-C 対応プログラム:3ECU
  各機能をランナブルとして実現しているプログラム．
  3個のECUで動作する．
  

３．フォルダ構成

	./os-applicatation  : AUTOSAR OS 対応プログラム
	./swc-applicatation : 
		./1ecu          : AUTOSAR SW-C 対応プログラム:1ECU
		./2ecu_co_b     : AUTOSAR SW-C 対応プログラム:2ECU
		./2ecu_cod_b    : AUTOSAR SW-C 対応プログラム:2ECU(機能追加)
		./2ecu_od_cb    : AUTOSAR SW-C 対応プログラム:2ECU(SW-C移動)
		./3ecu          : AUTOSAR SW-C 対応プログラム:3ECU
	./tools
		./SerialCommander.exe  : シリアルコマンダー(Windowsアプリケーション)
	./doc
		./モデルカー構築マニュアル.pdf
		./1.開発対象アプリケーション.pdf
		./2.開発環境.pdf

  
各AUTOSAR SW-C 対応プログラム に関しては，各フォルダの直下にECU毎のフ
ォルダが用意されている．

  
４．外部ソフトウェア

モデルカー制御プログラムを動作させるには，TOPPERSプロジェクトから公開
されている以下のソフトウェアが必要である．

ATK2-SC1   : HSBRH850F1L(CCRH)簡易パッケージ 2015-05-26
A-RTEGEN   : Release 1.2.2 
A-COMSTACK : Release 1.2.1
A-CANIF    : HSBRH850F1L(北斗電子)簡易パッケージ 2015-07-15

それぞれのファイルをダウンロードして，トップフォルダ(os-application等
が置かれているフォルダ)に展開すること．

ATK2-SC1に関しては，フォルダ名を atk2-sc1_1.3.2 から atk2-sc1 に変更す
ること．

autosar.org からダウンロードしたスキーマファイル
(AUTOSAR_4-0-3_STRICT.xsd, xml.xsd) を以下のフォルダにコピーすること．

・a-rtegen\bin\schema
・atk2-sc1\cfg\cfg



５. ハードウェア

モデルカー制御プログラムを動作させるためには，市販のRCカーと北斗電子製
のRCカーキット及びHSBRH850F1L100等を組み合わせたモデルカーを使用する．
HSBRH850F1L100は最小1台で動作し，最大3台までサポートする．

北斗電子 RCカーキット
 http://www.hokutodenshi.co.jp/7/HSBRH850F1L100.htm#rccar
 
北斗電子 HSBRH850F1L100
 http://www.hokutodenshi.co.jp/7/HSBRH850F1L100.htm
 
ランニングエレクトロニクス SBDBT5V
 http://www.runele.com/ca1/8/p-r-s/
 
PS3コントローラ

ルネサスエレクトロニクス E1
  http://japan.renesas.com/products/tools/emulation_debugging/onchip_debuggers/e1/
  
4.1 モデルカーの組み立て

RCカーキット取扱説明書(http://www.hokutodenshi.co.jp/download/manual/RCCAR_KIT_s.pdf)
及び，モデルカー構築マニュアルに従って組み立てる．

                
６．ビルド

AUTOSAR OS 対応プログラムのビルド方法に関しては，"2.開発環境.pdf"を参
照のこと．

AUTOSAR SW-C 対応プログラムのビルド方法は，次の通りである．

1.各フォルダの直下にある"ecu_extract.bat"を実行してECU Extractを実行す
  る．
  
2.各ECU毎のフォルダにおいて，"generator.bat"を実行して，RTEジェネレー
  タを実行する．
  
3.各ECU毎のフォルダにおいて，"configure.mtpj"をCS+でオープンして，ビル
  ド用のプロジェクト"atk2-sc1.mtpj"を作成する．
  
4.ビルド用のプロジェクトによってビルドを行う．


７．実行

実行方法については，"2.開発環境.pdf"を参照のこと．

PS3コントローラによる操作方法については，"1.開発対象アプリケーション
.pdf" を参照のこと．


7.1 パラメータ調整

ESC（スピードコントローラ）の種類によってはESCのパラメータ調整が必要な
場合がある．現状の動作確認済みはタミヤTLU-105BK/TEU-104BKであるが，同
じESCであっても個体差によりパラメータが異なる場合がある

詳細については，"2.開発環境.pdf"を参照のこと．
